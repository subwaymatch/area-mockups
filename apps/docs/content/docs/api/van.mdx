---
title: Van (cargo van)
description: A wrap-ready cargo van with a live livery panel on the cargo side.
---

`<Van>` and `<VanMockup>` render a generic Euro-style cargo van (a
Transit/Sprinter-class silhouette, no brand): the shell is the side profile —
clamshell hood, cowl break, raked windshield, high roof, wheel-arch cutouts —
extruded across the width, with wheels, A-pillar-raked door glass, a wide
slatted grille between headlights under the hood line, a license-plate
recess, a slim wrap-around bumper band, mirrors on long arms and hand-height
taillights added on. The flat cargo side carries a
live 900×368 vinyl-wrap panel for your livery. The mockup grounds the wheels
on a virtual road plane.

<ObjectDemo demo="van-basic" checker />

```tsx
<VanMockup streetSide={<Livery />} rear={<RearDoors />} deviceProps={{ rotation: [0, -0.5, 0] }}>
  <Livery />
</VanMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `streetSide` | `ReactNode` | - | Livery for the street-side (−Z) wrap panel. |
| `rear` | `ReactNode` | - | Livery for the rear-door panel, between the taillight clusters. |
| `color` | `string` | `'#eef0f2'` | Body paint. Wrap fleets are usually white. |
| `wrapBackground` | `string` | `'#ffffff'` | CSS background behind your wrap content. |
| `coverage` | `'panel' \| 'full'` | `'panel'` | `'full'` covers the entire side elevation and carves out the wheel arches, door glass, handle, mirror mount and door track, so the 3D hardware shows through the livery. |
| `resolution` | `number` | `900` panel · `1360` full | Virtual side-wrap width in CSS px; the rear panel shares its dpi. |

`children` is the curb-side (+Z) panel; with the default `coverage="panel"`
all three panels sit clear of the wheel arches, lights and cab glass. Rotate
the group to pose the van; the panels stay live at any angle.

## More examples

### Full-coverage wrap with carve-outs

`coverage="full"` stretches the live panel across the entire side elevation —
rockers to roofline, tail to nose. The panel is real DOM composited over the
WebGL canvas, so a CSS `clip-path` (computed from the same dimension spec the
geometry is built from) carves the body outline, wheel arches, door glass,
handle, mirror mount and door track out of it — the 3D wheels, windows and
hardware show through your livery, and pointer events over the carved regions
fall through to the orbit controls.

<ObjectDemo demo="van-full" />

```tsx
<VanMockup coverage="full" streetSide={<FullLivery />} deviceProps={{ rotation: [0, -0.42, 0] }}>
  <FullLivery />
</VanMockup>
```

### The rear three-quarter

Same van from behind: rear-door panel between the brake/turn/reverse
clusters, street-side wrap beyond.


### Painted body + full-color wrap

Match `color` and `wrapBackground` for a fully-wrapped look — the paint
carries across the cab, nose and roof that the panel doesn't cover.

<ObjectDemo demo="van-paint" />

```tsx
<VanMockup color="#1d4433" wrapBackground="#1d4433" autoRotate>
  <YourLivery />
</VanMockup>
```
