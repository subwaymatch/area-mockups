---
title: Brochure (tri-fold)
description: A standing letter Z-fold with three live panels, front and back.
---

`<Brochure>` and `<BrochureMockup>` render a US letter tri-fold standing in a
zig-zag (Z-fold) accordion. Each of the three 93×216 mm panels is its own
live 360×835 DOM surface — and the reverse faces can be printed too. Panels
shade gently toward the folds, like paper actually bowing.

Repeat the `Panel` slot — panels fill left to right in document order. Each
panel is independent live DOM.

<ObjectDemo demo="brochure-panels" checker />

```tsx
<BrochureMockup>
  <BrochureMockup.Panel><Front /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Middle /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Back /></BrochureMockup.Panel>
</BrochureMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `size` | `{ width?, height? }` | `93×216` | One panel's physical size in mm — e.g. `{ width: 99, height: 210 }` for an A4 tri-fold. |
| `foldAngle` | `number` | `24` | Zig-zag fold angle in degrees. `0` lays the sheet out flat. |
| `color` | `string` | `'#f5f4f0'` | Stock color of unprinted faces and paper edges. |
| `surfaceBackground` | `string` | `'#ffffff'` | CSS background behind each panel's content. |
| `resolution` | `number` | `360` | Virtual width of one panel in CSS px; height follows the panel aspect. |

The fold keeps every panel centered on the group origin, so the brochure
orbits about its own visual center at any `foldAngle`.

## Regions

The brochure has one repeating region:

| Slot | Surface |
| --- | --- |
| `<BrochureMockup.Panel>` | One panel, left to right in document order — up to three per side. Bare children are shorthand for the first (left) front panel. |

Add `side="back"` to a `Panel` slot to print a reverse face; back panels also
fill left to right in document order, as seen from BEHIND — matching how a
real tri-fold is imposed. Panels left out show bare `color` stock. Every slot
takes per-surface overrides — `background`, `resolution`, `interactive`,
`dragToRotate`, `style` — over the mockup-level defaults. The same slot
exists on the bare `<Brochure>`.

## More examples

### All six surfaces printed

A real tri-fold prints on both sides of the sheet, and every face here is a
live surface: three front panels, three with `side="back"` (reading left to
right as seen from the back). Spin the demo to read both sides.

<ObjectDemo demo="brochure-both-sides" />

```tsx
<BrochureMockup autoRotate>
  <BrochureMockup.Panel><Front /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Trails /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Visit /></BrochureMockup.Panel>
  <BrochureMockup.Panel side="back"><Map /></BrochureMockup.Panel>
  <BrochureMockup.Panel side="back"><Hours /></BrochureMockup.Panel>
  <BrochureMockup.Panel side="back"><Contact /></BrochureMockup.Panel>
</BrochureMockup>
```

### Flat, before folding

`foldAngle={0}` lays the sheet out flat — useful for reviewing the design as
the full 11" spread it prints as.

<ObjectDemo demo="brochure-flat" />

```tsx
<BrochureMockup foldAngle={0}>
  <BrochureMockup.Panel><Front /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Middle /></BrochureMockup.Panel>
  <BrochureMockup.Panel><Back /></BrochureMockup.Panel>
</BrochureMockup>
```

### One designed panel on bare stock

`children` is shorthand for the first panel; the rest show bare `color`
stock — a work-in-progress state that reads clearly in reviews.

<ObjectDemo demo="brochure-single" />

```tsx
<BrochureMockup color="#e9e4d8" float>
  <Front />
</BrochureMockup>
```
