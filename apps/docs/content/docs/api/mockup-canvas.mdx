---
title: MockupCanvas
description: The ready-made stage, with a WebGL canvas, studio lighting, soft shadows, and drag-to-rotate controls.
---

`<MockupCanvas>` wraps a react-three-fiber `Canvas` with everything a product
shot needs. Compose it with any device, for example
`<MockupCanvas><Phone>...</Phone></MockupCanvas>`.

One canvas, two bare devices, one shared studio light rig and shadow plane.

<ObjectDemo demo="phone-duo" checker />

```tsx
<MockupCanvas camera={{ position: [0, 0.5, 8.6], fov: 40 }} shadowY={-2.35}>
  <Phone variant="s26ultra" position={[-1.6, 0, -0.2]} rotation={[0, 0.3, 0]}>
    <YourAndroidApp />
  </Phone>
  <IPhone variant="promax" position={[1.6, 0, 0]} rotation={[0, -0.3, 0]}>
    <YourIOSApp />
  </IPhone>
</MockupCanvas>
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | Your scene, typically a device such as `Phone`. |
| `controls` | `boolean` | `true` | Drag-to-rotate controls with the axis fixed at the stage center (pan is always disabled). Vertical rotation stays within the classic orbit clamp by default. |
| `freeRotation` | `boolean` | `false` | Allow the camera to rotate a full 360° vertically — vertical drags continue straight over the top and bottom of the device instead of clamping. |
| `autoRotate` | `boolean` | `false` | Slowly spin the camera around the device. |
| `autoRotateSpeed` | `number` | `1` | Spin speed when `autoRotate` is on. |
| `zoom` | `boolean` | `false` | Pinch zoom on touch, scroll-wheel zoom on desktop, plus overlay +/− buttons. Off by default so embedded mockups never hijack page scroll; on, the canvas owns two-finger gestures. |
| `fullscreen` | `boolean` | `false` | Adds an overlay button that expands the mockup to fill the screen via the Fullscreen API (and collapses it back). Off by default. |
| `shadows` | `boolean` | `true` | Soft contact shadow under the device. |
| `shadowY` | `number` | `-2.05` | Y position (world units) of the shadow plane. Grounds the device on its shadow. The mockup wrappers compute this per device. |
| `environment` | `boolean` | `true` | Procedural studio lighting and reflections, generated on the GPU with no HDR downloads. |
| `background` | `string` | - | CSS background of the canvas (color, gradient, or image). Transparent when unset — that is what the checkerboard behind each page's first example is showing through. |
| `camera` | r3f camera | `[0, 0.5, 7.4]`, fov 40 | Override the default camera. |
| `dpr` | `number \| [min, max]` | `[1, 2]` | Device pixel ratio clamp. Keeps hi-dpi GPU load predictable. |
| `className`, `style` | - | - | Forwarded to the canvas wrapper. |

## More examples

### Zoom controls

`zoom` enables pinch on touch, the scroll wheel on desktop, and overlay
+/− buttons on both. Zooming clamps to the same distance range as the
orbit controls.

<ObjectDemo demo="canvas-zoom" />

```tsx
<BookMockup zoom float color="#4a2c3f">
  <YourCoverArt />
</BookMockup>
```

### Full-screen view

`fullscreen` adds an overlay button that expands the mockup to fill the whole
screen through the browser's Fullscreen API, with a matching button to collapse
it back. It composes with `zoom`, and works on any mockup.

```tsx
<BookMockup fullscreen float color="#4a2c3f">
  <YourCoverArt />
</BookMockup>
```
