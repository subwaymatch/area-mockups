title: Book (hardcover)
description: A trade hardcover with live front cover, back cover and spine.
---

`<Book>` and `<BookMockup>` render a 156×234 mm trade hardcover: cloth-wrapped
binder's boards with a rounded flush spine and case-binding joint groove, a
cream page block recessed behind the board overhang on the three open edges,
and live DOM on the front cover, the back cover, and a spine strip riding
the backbone crown.

Any React node fills the front board, full bleed. Match `color` to the
jacket's palette so the spine and back board feel like the same book.

<ObjectDemo demo="book-basic" checker />

```tsx
<BookMockup color="#16324a" rotation={[0, 0.35, 0]}>
  <YourCoverArt />
</BookMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `size` | `{ width?, height?, thickness? }` | `156×234×27` | Physical trim in mm — e.g. `{ width: 216, height: 279 }` for a letter-size art book, `{ thickness: 45 }` for a fat novel. |
| `color` | `string` | `'#1f3a5f'` | Cloth color: spine, back board, and board edges. |
| `pageColor` | `string` | `'#f4eede'` | Paper color of the page block edges. |
| `surfaceBackground` | `string` | `'#ffffff'` | CSS background behind your cover content. |
| `resolution` | `number` | `480` | Virtual cover width in CSS px; height follows the board aspect. |

Every example below is live: drag to spin it.

## Regions

| Slot | Surface |
| --- | --- |
| `<BookMockup.Cover>` | Front cover (bare children) — fills the front board, full bleed. |
| `<BookMockup.Back>` | Back cover — fills the back board, full bleed. |
| `<BookMockup.Spine>` | Spine — a tall strip on the backbone crown. |

Every slot takes per-surface overrides — `background`, `resolution`,
`interactive`, `dragToRotate`, `style` — over the mockup-level defaults. The
same slots exist on the bare `<Book>`.

## More examples

### Every surface printed

The `.Back` and `.Spine` slots complete the case: blurb and barcode on the
back board, the title reading down the spine — the whole book stays designed
as it turns.

<ObjectDemo demo="book-full" />

```tsx
<BookMockup autoRotate color="#16324a">
  <YourCoverArt />
  <BookMockup.Back>
    <BackBoard />
  </BookMockup.Back>
  <BookMockup.Spine>
    <SpineStrip />
  </BookMockup.Spine>
</BookMockup>
```

### Floating and auto-orbiting

The same idle animations every device mockup has. `pageColor` warms up the
page block for an older, cream-paper look.

<ObjectDemo demo="book-float" />

```tsx
<BookMockup color="#3d2352" pageColor="#efe8db" autoRotate float>
  <YourCoverArt />
</BookMockup>
```

### A row of books — composing bare objects

Skip the all-in-one wrapper and drop several `<Book>` objects into one
[`MockupCanvas`](/docs/api/mockup-canvas) — a series lineup in a single
WebGL context.

<ObjectDemo demo="book-row" />

```tsx
<MockupCanvas camera={{ position: [0, 0.4, 9.4], fov: 40 }} shadowY={-2.2}>
  <Book color="#16324a" position={[-3, 0, -0.4]} rotation={[0, 0.5, 0]}>
    <CoverOne />
  </Book>
  <Book color="#5e2431" position={[0, 0, 0]} rotation={[0, 0.15, 0]}>
    <CoverTwo />
  </Book>
  <Book color="#274233" position={[3, 0, -0.4]} rotation={[0, -0.35, 0]}>
    <CoverThree />
  </Book>
</MockupCanvas>
```
