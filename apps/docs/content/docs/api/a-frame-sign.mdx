`<AFrameSign>` and `<AFrameSignMockup>` render the classic 600×900 mm
chalkboard-class sidewalk sign: two wood-framed panels hinged at the top,
legs splayed ~15° (measured from street photos), with both faces live DOM —
chalk menus, sale boards, open/closed signs.

<ObjectDemo demo="aframe-basic" checker />

```tsx
<AFrameSignMockup rotation={[0, -0.3, 0]}>
  <MenuBoard />
</AFrameSignMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `color` | `string` | `'#4a3826'` | Frame color — dark-stained wood by default. |
| `surfaceBackground` | `string` | `'#20241f'` | CSS background behind each panel (chalkboard black). |
| `resolution` | `number` | `420` | Virtual face width in CSS px; height follows the panel. |

## Regions

| Slot | Surface |
| --- | --- |
| `<AFrameSignMockup.Front>` | Front panel (bare children). |
| `<AFrameSignMockup.Back>` | Back panel — the other side of the A. |

Every slot takes per-surface overrides — `background`, `resolution`,
`interactive`, `dragToRotate`, `style` — over the mockup-level defaults. The
same slots exist on the bare `<AFrameSign>`.

## Mockup-able areas

Every live surface filled with chroma green, labeled with the slot that
