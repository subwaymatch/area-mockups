`<Phone>` renders a Galaxy device; `<PhoneMockup>` is the all-in-one version.
Both variants keep true relative sizes and per-model camera architecture — the
6.3″ Galaxy S26 and the 6.9″ Galaxy S26 Ultra, with detail geometry (button
pills, camera island, port cutouts, antenna seams) measured from scans of the
retail hardware.

<ObjectDemo demo="phone-basic" checker />

```tsx
<PhoneMockup color="#15171d" frameColor="#4d5260" deviceProps={{ rotation: [0, -0.25, 0] }}>
  <TapCounter />
</PhoneMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content) and, on the mockup, the
[shared mockup props](/docs/api):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `'s26' \| 's26ultra'` | `'s26'` | Which Galaxy device: true relative sizes, per-model cameras (S26 triple-lens pill island, Ultra large proud rings + tele column and S Pen silo), and logical resolutions. |
| `colorway` | `string` | — | A retail colorway id from `GALAXY_COLORWAYS` (S26: `navy`, `cobaltviolet`, `icyblue`, `mint`, `silvershadow`; Ultra: `titaniumsilverblue`, `titaniumblack`, `titaniumgray`, `titaniumwhitesilver`). Presets `color` + `frameColor`; explicit color props override it. |
| `orientation` | `'portrait' \| 'landscape'` | `'portrait'` | Landscape lays the device on its side and swaps the virtual display (for example 780×360 on the S26) with upright content. |
| `color` | `string` | `'#101216'` | Back panel colorway. |
| `frameColor` | `string` | `'#4a4f59'` | Metal frame, buttons, and camera rings. |
| `punchHole` | `boolean` | `true` | Front camera punch hole overlay. |
| `resolution` | `number` | per variant | CSS pixel width of the virtual display; height follows the panel aspect (360 gives 360×780 on the S26). |

## Colorways

Try Icy Blue `#c9d6e8`, Navy `#2c3344`, Mint `#c8e6d2`, or Marble Gray
`#d3d6dd` with `frameColor="#b6bac4"`. The Ultra's Titanium Silverblue is
`color="#a9bdce"` with `frameColor="#c2ccd7"`.

## More examples

Every example is live: drag to spin it, and the screens really work.

### Colorways with float and auto-orbit

<ObjectDemo demo="phone-lock" />

```tsx
<PhoneMockup autoRotate float color="#d3d6dd" frameColor="#b6bac4" screenBackground="#000">
  <LockScreen />

```

### Landscape, at the Ultra's real resolution

`orientation="landscape"` swaps the virtual display to 832×384 on the
S26 Ultra — your responsive layout reacts exactly like on the device.

<ObjectDemo demo="phone-landscape" />

```tsx
<PhoneMockup variant="s26ultra" orientation="landscape" color="#2e3238" frameColor="#565b64">
  <MusicPlayer />
</PhoneMockup>
```

### Galaxy S26 — the vertical pill camera island

`variant="s26"` renders the 6.3″ design: the three lenses live in a single
vertical pill island with the flash on the flat back beside it.

<ObjectDemo demo="phone-s26" />

```tsx
<PhoneMockup variant="s26" color="#6f6791" frameColor="#5a5478">
  <MusicPlayer />
</PhoneMockup>
```

### Galaxy S26 Ultra

`variant="s26ultra"` renders the 6.9″ flagship: boxier corners, three proud
rings on the pill island with the tele column beside it, the S Pen silo, and a
384×832 virtual screen.

```tsx
<PhoneMockup variant="s26ultra" color="#a9bdce" frameColor="#c2ccd7">
  <MusicPlayer />
</PhoneMockup>
```
