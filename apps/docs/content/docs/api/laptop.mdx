`<Laptop>` and `<LaptopMockup>` render an Apple-style laptop: unibody deck
with the real 78-key Magic Keyboard layout (printed legends, rendered as a
single instanced draw call), trackpad, hinge band, ports, and a notched
display. Two variants at true relative size: the MacBook Air 13" (M5) and the
MacBook Pro 14" (M5) — the Pro adds its scan-accurate port array (MagSafe,
2× Thunderbolt and headphone jack left; HDMI, Thunderbolt and SDXC right),
perforated speaker strips flanking the keyboard, bigger feet, the lid badge
and the embossed underside wordmark.

The window on screen is real DOM — click it, drag the mockup.

```tsx

  <YourDesktopApp />
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `'air13' \| 'pro14'` | `'air13'` | MacBook Air 13" or MacBook Pro 14" — true relative sizes, per-model ports, feet, speaker grilles and notch. |
| `colorway` | `string` | — | A retail colorway id from `LAPTOP_COLORWAYS` (Air: `silver`, `starlight`, `midnight`, `skyblue`; Pro 14: `spaceblack`, `silver`). Presets `color`; an explicit `color` prop overrides it. |
| `color` | `string` | `'#e3e4e6'` | Aluminum finish. Try Sky Blue `#aec6d9`, Starlight `#e8e0d4`, Midnight `#2e3642`, or Space Black `#4a484b` on the Pro. |
| `notch` | `boolean` | `true` | Camera notch overlay at the top of the display. |
| `openAngle` | `number` | `110` | Lid angle in degrees between deck and screen (90 is upright). |
| `resolution` | `number` | per variant | Virtual display width; the Air defaults to 1280 (1280×832) and the Pro 14 to 1512 (1512×982) — exactly the default scaled desktop resolutions, so desktop breakpoints apply. |

There is no `orientation` prop; laptops only have one pose.
