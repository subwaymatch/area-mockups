`<Magazine>` and `<MagazineMockup>` render a 216×279 mm letter-trim,
perfect-bound magazine: a thin page block with crisp trimmed paper edges on
the three open sides, the glossy cover stock wrapping the bound spine, and
live 480×620 DOM front and back covers, each under a soft cover-gloss
sheen.

Real editorial CSS lays out exactly as it would in a browser at 480×620.

<ObjectDemo demo="magazine-basic" checker />

```tsx
<MagazineMockup deviceProps={{ rotation: [0, -0.3, 0] }}>
  <YourCoverArt />
</MagazineMockup>
```

## Props

Beyond the [shared screen props](/docs/screen-content):

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `back` | `ReactNode` | - | Back cover design — full bleed, same glossy sheen. |
| `size` | `{ width?, height?, thickness? }` | `216×279×6` | Physical trim in mm — e.g. `{ width: 210, height: 297 }` for A4, `{ thickness: 12 }` for a thick issue. |
| `pageColor` | `string` | `'#fbfaf7'` | Paper color of the trimmed page edges. |
| `backColor` | `string` | `'#e9e7e2'` | Back cover and spine-wrap color. |
| `glossy` | `boolean` | `true` | Soft diagonal sheen over the cover art (the glossy stock look). |
| `coverBackground` | `string` | `'#ffffff'` | CSS background behind your cover content. |
| `resolution` | `number` | `480` | Virtual cover width in CSS px; height follows the trim aspect. |

## More examples

### The back-cover ad

Magazines sell their back page — `back` prints it, gloss sheen and all.

<ObjectDemo demo="magazine-back" />

```tsx
<MagazineMockup autoRotate back={<BackCoverAd />}>
  <YourCoverArt />
</MagazineMockup>
```

### Matching the stock to the issue

`backColor` paints the back cover and the perfect-bound spine wrap;
`pageColor` tints the trimmed page edges. Dark issues look best when all
three agree.

<ObjectDemo demo="magazine-stock" />

```tsx
<MagazineMockup pageColor="#efeadd" backColor="#101319" float>
  <YourCoverArt />
</MagazineMockup>
```
